- **회의 일시:** 2025년 11월 7일 (금) 오후 4:06 ~ 4:33 (27분간)

---

## 2. 참석자 명단

---

## 3. 주요 안건
1. senko 및 Whisper 결합 모델의 중복 발화 문제 검증  
2. 전처리 방식(VAD, STFT, 노이즈 제거 등)별 성능 비교 및 개선 방향 논의  
3. STT 인식 정확도 향상 및 이름 인식 문제 검토  
4. 향후 실험 및 주말 과제 분담  

---

## 4. 회의 내용

### 🔹 모델 검증 결과 공유
- **senko + Whisper 조합 실험:**  
  - Whisper Large 모델 사용 시 18분, Turbo 사용 시 6분 소요.  
  - 동일 구간 발화가 **중복으로 분리 표시되는 오류** 발생(예: 10초~16초 발화가 스피커 4와 2로 중복).  
  - 중복 원인은 senko 내부 세그먼트 처리 문제로 추정됨.  
  - 클로바노트 결과와 비교 시, senko는 발화자 구분이 과도하게 나뉘는 반면, 클로바는 한 명으로 통합 처리.  

- **결론:** senko는 속도는 빠르나 **중복 분리 문제 존재**, 발화자 분리 정확도 불안정.  

---

### 🔹 Whisper 모델 비교
- Whisper Large vs Turbo 성능 비교 결과 **Large가 더 안정적**.  
- Turbo는 속도는 빠르지만 문장 분리 오류 및 누락 발생.  
- Whisper Large v3 사용 시 STT 품질 및 화자 일관성 개선 확인.  
- 노이즈 환경에서는 여전히 성능 하락 발생.  

---

### 🔹 전처리 실험 결과
- **참석자 4:**  
  - 노이즈 제거 파이프라인 구성: STFT, Mel-Spectrogram, VAD 세 모듈로 분리 실험.  
  - DC offset 제거 및 평균 파형 조정 후, VAD 적용 시 잡음 제거는 우수했으나 **말 겹침 현상 발생**.  
  - 내부 VAD 중복 적용 문제로 **VAD 단독 제거 후 진행** 제안.  
  - STFT + Mel-Spectrogram 조합으로 **잡음 제거 및 발음 명확화 효과 확인**.  

- **참석자 2:**  
  - NVIDIA Nemo 설정 파일(Config) 직접 수정으로 성능 향상 확인.  
  - 민감도 조절 및 군집화 임계값 조정 시 회의 음성에서는 **우수한 성능**을 보임.  
  - 단, 과적합 가능성 존재(회의 데이터에만 최적화된 결과).  

---

### 🔹 이름 인식 및 태깅 방식 논의
- STT 모델에서 이름 인식 오류 다수 발생 (예: “민서” → “인서”, “재영” → “재훈”).  
- **AI 자동 태깅 vs 사용자 수동 입력** 간 UX 논의:  
  - 사용자 직접 발화(“저는 ○○입니다”) 방식은 현실성 낮음.  
  - 대신 **화자별 말투·주제 기반 추정 후 사용자 최종 확인 구조** 제안.  
  - 예: “재훈으로 추정되는 화자 A는 찬성 의견을 주로 냄” 형태로 자동 분석 → 사용자가 클릭으로 검증.  
  - 시각화 아이디어: **워드 클라우드 기반 화자별 발화 요약 UI** 제안.  

---

### 🔹 모델 평가 및 주말 과제
- **모델별 평가 요약:**  
  - **Nemo:** 가장 안정적, STT-화자 태깅 연동 성능 우수.  
  - **senko:** 빠르지만 불안정, 중복 분리 문제 지속.  
  - **Assembly AI:** 품질 양호, 빠르지만 포트폴리오 구현 제약.  

- **주말 과제:**  
  1. 전처리(노이즈 제거) 방식 고도화 및 통일화(참석자 4 담당).  
  2. STT 성능 향상 및 Whisper+Nemo 결합 실험(김민서 담당).  
  3. 화자 태깅 방식 설계 및 UX 개선 아이디어 구체화(참석자 2·5 담당).

## 5. 채택사항과 유보사항
- **채택사항**
  - Whisper Large 기반 STT 채택, Turbo는 보류.  
  - VAD 제거 및 STFT+Mel-Spectrogram 전처리 방식 채택.  
  - 화자 태깅 구조는 AI 추정 + 사용자 검증 방식으로 방향 확정.  
  - STT·태깅 분리 후 파이프라인 통합 계획 유지.  

- **유보사항**
  - STT 이름 인식률 개선(‘민서→인서’ 등).  
  - 사용자 입력 최소화 UX 방식 구체화.  
  - senko 중복 분리 문제 재검증 필요.
