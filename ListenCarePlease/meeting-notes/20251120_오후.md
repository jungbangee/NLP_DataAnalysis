### AI 도구 활용 및 문제 해결

- 노션 AI 도입 및 테스트 진행
    - 노션 AI를 활용해 테스트를 완료하고 문제점 점검
- 슬라이더 연동 및 알짜 기준 문제 해결
    - R 값 변동 문제는 시드값 미지정으로 발생했으며 시드값 지정으로 해결
- ERM 외 정규 표현식, 키워드 매칭, 고문 분석 방법 비교
    - 여러 방법을 시도했으나 ERM 방식이 가장 효과적임 확인
- 날짜 처리 방식 논의
    - 현재 날짜를 하드코딩으로 처리 중이며, 파일명에 날짜 명시 또는 입력 방식 도입 검토
    - 사용자에게 날짜 명시를 요구하는 방안에 찬성 의견 다수
- UI 연동 및 코드 수정 계획
    - 날짜 처리 방식 확정 후 UI 및 코드 수정 예정

### 회의 템플릿 설계

- 템플릿 분류 기준 및 구조 설계
    - 회의 템플릿을 정보 교류용, 문제 해결용, 협업용 세 가지로 분류
    - 각 템플릿은 배타적이지 않으며 상황에 따라 중복 적용 가능
- 템플릿 내 항목 중복 및 범주화 문제 인지
    - 템플릿 항목이 겹칠 수 있어 배타적 구분은 어려움
- 향후 템플릿별 구조, 배치, 글자 수 조정 계획
    - 각 템플릿을 구체적으로 시도하며 개선 예정

### 정량적 지표 시각화 및 데이터 분석

- 지표 선정 완료 후 시각화 테스트 진행
    - 도넛 차트, 바 차트 등으로 발화량, 점유율, 발화 빈도 시각화
- 발화 속도 및 발화량 분석
    - 시간 대비 단어 수로 발화 속도 판단 가능
    - 발화 기회별 평균 발화 시간 및 빈도 산출
- 침묵 구간 과다 산출 문제 발생
    - 음성 파일 메타데이터와 다이얼제이션 결과 불일치로 긴 침묵 구간 산출
    - 다이얼제이션 세그먼트 기준으로 발화 시간 재정의 방안 검토
- TTR(총 단어 대비 고유 단어 비율) 계산 및 해석
    - 형태소 분석으로 명사 추출 후 슬라이딩 윈도우 방식으로 TTR 산출
    - 단어 재사용 정도 파악 가능하나, 일상어 포함으로 지표 설득력 제한
    - 형태소 분석 처리 시간이 24분 이상 소요되는 문제점 존재

### LLM 모델 비교 및 오탈자 처리

- LLM 사용 여부 및 모델별 성능 비교
    - LLM 미사용 시 처리 방식과 LLM 사용 시 요약 방식 구분
    - 엑사원, QWEN 모델 테스트 결과 각각 장단점 존재
    - 엑사원은 문장 반복 및 요약 길이 문제, QWEN은 신조어 처리 미흡
- 프롬프트 수정 및 추가 모델 테스트 계획
    - 프롬프트 개선 시도 중이며 잼마, 솔라라, 미스트랄 등 로컬 모델 테스트 예정
- 오탈자 처리 필요성 및 위치 논의
    - 오탈자 처리는 후순위로 두고 앞선 단계에서 처리하는 방안 검토
    - 오탈자 과도 처리 시 단어 추출 오류 우려
    - STT 성능은 양호하나 오탈자 영향은 존재
    - 완벽한 오탈자 수정보다 적정 수준 리스크 감수 필요성 공감
- 오탈자 처리 시간과 품질 간 트레이드오프 인식
    - 품질 향상을 위한 시간 소요가 사용자 경험에 부정적 영향 가능성
