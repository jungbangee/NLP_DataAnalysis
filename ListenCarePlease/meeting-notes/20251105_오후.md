## 4. 회의 내용

### 🔹 파이프라인 개념 정리

- 팀 내 일부가 **"음성 데이터 학습이 필요한가"**에 대해 혼동이 있었음.
- 합의 결과, **STT는 학습하지 않고 오픈소스 모델을 활용**, 이후 **화자 분리 및 식별 단계에 집중**하기로 결정.
- 텍스트만으로는 화자 구분이 어렵기 때문에, **음성 특징(스펙트로그램, MFCC 등)을 함께 학습**하는 멀티태스크 구조로 접근 필요성이 제기됨.

---

### 🔹 모델 구성 방향

- **STT(음성 → 텍스트):**
    - **Whisper** 모델을 기본 옵션으로 사용.
    - 학습 불필요, 성능 검증 후 바로 적용 예정.
    - 한국어 특화 모델(네이버 클로바 스피치, 비토, ETRI API 등)은 후보군으로 남김.
- **화자 분리(Speaker Diarization):**
    - **Pyanote.audio**(오픈소스 화자 분리 모델) 도입 검토.
    - Whisper → Pyanote 순서로 STT 후 화자 분리 또는
    Pyanote → Whisper 순서(선분리 후 인식) 중 최적 구조 비교 후 결정 예정.
    - 분리와 식별을 통합한 **멀티태스킹 모델 설계 가능성** 제안됨.
- **화자 식별(Speaker Identification):**
    - 각 화자의 고유 음성 특징 벡터를 추출하여 DB에 저장 후 비교하는 구조 제안.
    - STT 결과와 연동하여 **"화자 A – 발화 내용"** 형태의 태깅 데이터 자동 생성 목표.

---

### 🔹 학습 및 서비스 구조 논의

- 파인튜닝은 도메인 특화 모델이 아니므로 **필요하지 않음**으로 의견 일치.
- 모델은 일반화된 환경에서 작동해야 하므로 **기존 학습된 가중치 활용** 방침.
- 1차 목표: **이름 없이 화자 구분이 가능한 형태('화자1', '화자2' 식)**의 결과물 산출.
- 2차 확장: **화자 이름 매핑 및 요약 기능 결합**을 통한 회의록 자동화 서비스 확장.

---

### 🔹 응용 아이디어 논의

- 회의록 요약, 뉴스/방송/강연 등 다인 대화 음성 자동 태깅,
스포츠·인터뷰 영상의 발화자 구분 등 다양한 서비스 가능성 논의.
- 카카오톡 대화 요약형 AI 기능을 참고한 **인물별 요약 시스템** 구상도 제안됨.

## 5. 채택사항과 유보사항

### 채택사항
- Whisper + Pyanote 조합으로 STT 및 화자 분리 진행.
- 파인튜닝은 생략하고 사전학습 모델 활용.
- 이번 주 내에 화자 분리(기본 형태) 결과 도출 목표 설정.

### 유보사항
- Pyanote 오디오의 신뢰성과 한국어 음성 적합성 검증.
- STT → 분리 vs 분리 → STT 순서 중 최적 구조 확정.
- 데이터베이스 구조 및 화자 이름 매핑 방식 세부 결정.
