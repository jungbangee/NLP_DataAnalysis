## 4. 회의 내용

### 🔹 모델 성능 및 비교
- **구글 클라우드 STT:**  
  - 접근 권한 및 과금 이슈로 진행 중단.  
  - 대용량 파일(15분 이상) 처리 시 시간 지연 및 비용 발생.  

- **NVIDIA Nemo:**  
  - 기본 성능 양호하나, 전처리 수준에 따라 결과 편차 큼.  
  - 노이즈 제거 시 4명 화자까지 안정적으로 분리 가능.  
  - 전처리 알고리즘을 라이트하게 구성 필요.  

- **SpeechBrain:**  
  - 15분 파일 기준 처리 속도 빠름(0.3~0.5초 단위 타임스탬프).  
  - 화자 수 확장 가능하나 STT 결과와의 싱크 불안정.  

- **Assembly AI:**  
  - STT + 화자 분리 통합형 API로 간단한 사용성.  
  - 5명까지 화자 식별 가능하나, 비용 발생(기본 $50 크레딧).  
  - 직접 개발 요소가 적어 포트폴리오 측면에서 활용도 낮음.  

- **Senko :**  
  - Pyannote 기반 모델로, 속도 매우 빠름(L4 기준 7.8초, T4 기준 15초).  
  - 화자 분리 정확도 우수하며, 세그먼트 단위(`speaker, start, end`) 구조로 출력.  
  - 한국어 성능은 추가 검증 필요.

### 🔹 모델 활용 방향 및 검증 계획
- Pyanote는 후보군에서 제외, Nemo, SpeechBrain, Seiko, Assembly AI 중심으로 검증 진행.  
- 내일(11월 7일) 점심 시간 중, **소음이 많은 환경(식사 중·카페 등)** 에서 실제 음원 녹음 테스트 진행.  
- **노이즈 환경에서도 화자 분리 정확도를 유지하는 모델**을 최종 후보로 선정 예정.  
- “챌린징 환경” 검증을 통해 모델의 실사용 안정성 판단.  

---

### 🔹 서비스 방향 및 파이프라인 논의
- **핵심 목표:** 화자 분리 및 태깅 기반 회의록, 자막, 요약 서비스 구축.  
- **추가 기능 제안:**  
  - 회의록 요약 및 자동 자막 생성(넷플릭스·예능 등 미디어 응용 가능).  
  - 화자 이름 태깅 부재 시 **AI 기반 별칭 생성 또는 사용자가 직접 입력 가능한 웹 UI 설계** 논의.  
  - 자막 서비스 확장 시 엔딩 크레딧 자동 인식 기능 또는 외부 출연진 정보 크롤링 검토.  
- **파이프라인 방향:**  
  - 음성 입력 → 화자 분리 → STT 변환 → 화자 태깅 → 자막/요약/번역 → 시각화 출력.  
  - 실시간 번역은 난이도 높아 기획서 반영 단계에서 보류.
