## 4. 회의 내용

### 🔹 투두리스트 관리
- 강사 피드백에 따라 **매우 세부적인 투두리스트 작성** 요구.
- 각자 이름별로 페이지를 만들어 일별 수행 내역과 계획을 구체적으로 작성.
- 강사가 매일 직접 확인 예정으로, **진행상황을 엄격히 관리**.

---

### 🔹 주제별 탐색 결과 공유

#### ① NLP용 로우데이터 구조화 AI
- **설명:** 웹페이지 전체를 OCR로 이미지화 후, 바운딩 박스 단위로 구조화.
- **문제점:** 기술 복잡도가 높고 OCR 기반 학습이 어렵다는 점 확인.
- **평가:** 기간 내 수행이 어려움. 아이디어는 신선하나 실현성 낮음.

#### ② 세계 경제 뉴스 요약 AI
- **데이터 출처:** 누리집 등 한국어 뉴스 데이터, 위키독스의 '뉴스 요약봇' 참고.
- **구상:** Summarization + Translation 결합 모델 제안 (미국 뉴스 → 한국어 요약).
- **기술 논의:**
  - 도메인 차이(경제 뉴스 vs 상품 리뷰)는 LLM 파인튜닝으로 보완 가능.
  - 다국어 모델(multilingual LLM) 활용 방안 검토됨.
  - 실시간 번역·요약을 통한 **경제 정보 전달 지연 단축** 아이디어 제시.
- **평가:** 데이터 접근성 양호, **서비스 확장성 높음**.

#### ③ 보이스피싱 탐지 AI
- **데이터 출처:** 금감원·KR CCVI에서 확보한 음성 데이터 약 1,200건.
- **문제점:** 데이터 양이 부족하며, STT 변환 과정이 매우 시간 소모적.
- **평가:** 데이터 확보 어려움으로 **보류 검토 중**.

#### ④ 작가 문체 변환 AI
- **데이터 출처:**
  - 캐글 'Books Author Classic' (영문)
  - AI 허브 '대규모 도서 기반 한국어 말뭉치' (국문)
- **논의 내용:**
  - 문체 분석은 NLP의 심화 테스크로, 문학·비문학 확장 가능성 있음.
  - 한국어 중심의 데이터 확보 필요.
  - 평가 기준(문체 유사성 판단)이 명확하지 않다는 점이 과제로 지적됨.
- **평가:** 포트폴리오 가치 높으나 **평가체계 보완 필요**.

#### ⑤ 음성 기반 자동 태깅 요약 서비스
- **핵심 아이디어:** 화자 식별(Speaker Identification) + STT 기반 자동 회의록 태깅.
- **기술 구성:**
  - 화자 인식 → 화자 검증 → 화자 분리 단계 이해 공유.
  - AI 허브의 한국어 화자 인식용 음성 데이터 활용 가능.
  - 녹음 전 "자기소개 음성" 입력을 통해 사전 학습된 화자 벡터 생성 방안 제시.
- **평가:** 음성 AI와 NLP 융합형 서비스로서 실용성 높음.

### 🔹 전반 논의 요약
- **보이스피싱 AI:** 데이터 부족으로 실현성 낮음.
- **작가 문체 변환 AI:** 난이도 높지만 차별화된 주제, 포트폴리오 가치 큼.
- **세계 경제 뉴스 요약 AI:** 현실적인 실행 가능성과 응용 확장성 모두 높음.
- **자동 화자 태깅 서비스:** 기술·서비스적 확장 가능성 큼, NLP+Audio 융합으로 평가.

## 5. 채택사항과 유보사항

### 채택사항
- 투두리스트를 일 단위로 세부 관리.
- 내일 오전 회의에서 주제별 프로포절 발표 진행.
- 각자 최소 1개 이상의 데이터셋 확보 필수.

### 유보사항
- 최종 주제 확정 (내일 오전 회의에서 결정 예정).
- 작가 문체 AI의 평가 체계 및 검증 방법 보완.
- 보이스피싱 AI의 데이터 확충 방안 추가 조사 필요.
